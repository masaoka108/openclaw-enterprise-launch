OpenClaw企業導入支援サービス 料金設計（最終補正版）

更新日: 2026-03-17
用途: 提案・社内説明・見積の基準

1. 価格設計の前提

OpenClaw の法人導入は、単なるAIツール提供ではなく、対象業務の選定、導入設計、ガバナンス整備、現場定着、継続改善まで含めて成果が決まります。
したがって価格は「ライセンス費」ではなく、以下3つの価値に対する対価として設計します。

  1. 初期導入費: 課題整理、要件定義、環境設計、初期教育
  2. 月額運用支援費: 定着支援、改善提案、レビュー、問い合わせ対応
  3. 個別開発費: システム連携、個社ワークフロー、監査/権限要件対応

2. 市場での相対ポジション

OpenClaw の狙いは、安価なPoC支援会社や単体SaaSより上、SIerの大規模個別開発よりは下のポジションです。

  • 下位帯: 単発PoC、研修中心、月額20〜40万円前後
  • OpenClawの本命帯: 導入設計 + 定着伴走 + 標準連携まで含む、月額80〜150万円帯
  • 上位帯: 大規模SI/フルスクラッチ/複雑な統合案件、初期数百万円〜数千万円

このため、価格は**「安いから選ばれる」ではなく、「導入をちゃんと前に進められる価格」**に置くべきです。

3. 避けるべき安売りゾーン

以下は避けるべき価格帯です。

  • Pilot を月額30万円未満にする
  • Business を月額60万円未満にする
  • Enterprise を月額120万円未満 / 初期300万円未満で見せる
  • 値引きで調整し、運用レビュー・設計工数を削れなくなる

理由は明確です。

  • 導入設計と定着支援の工数を吸収できない
  • 「高い専門性を伴う導入支援」ではなく「安い運用代行」に見えてしまう
  • Enterprise の重みが薄れ、Business との線引きが曖昧になる
  • 営業は取りやすく見えても、継続率と粗利が崩れる

値引きではなく、対象部門数、定例頻度、連携範囲、SLAで調整するのが原則です。

4. 100〜1,000名企業の予算許容度を踏まえた推奨プラン

100〜1,000名規模の中堅企業では、AI導入の本格検討において以下のレンジが通しやすい水準です。

  • 初期費用: 100万〜400万円台
  • 月額: 50万〜120万円台
  • 初年度総額: 1,000万〜3,000万円前後（支援 + 一部連携込み）

この前提を踏まえ、OpenClaw の推奨価格は以下とします。

Pilot / 部門導入プラン

  • 初期導入費: 60万円
  • 月額: 35万円
  • 契約期間目安: 3〜6か月
  • 対象: 1部署でのPoC〜初期定着

含まれる内容

  • 課題整理ワークショップ 1回
  • ユースケース選定 2〜3件
  • 基本設定 / 権限設計
  • 標準連携の初期設定
  • 月1回レビュー
  • メール / チャットサポート

位置づけ

  • 入口商品だが、安売り商品ではない
  • 「試す」ではなく、次の本格展開判断に足る成果を作るためのプラン

Business / 本格運用プラン

  • 初期導入費: 150万円
  • 月額: 80万円
  • 契約期間目安: 6〜12か月
  • 対象: 複数部門展開・運用定着・全社展開準備

含まれる内容

  • 導入ワークショップ 2〜3回
  • 業務要件整理 / 導入ロードマップ作成
  • 主要SaaS連携の設定支援
  • 利用ルール / ガイドライン整備支援
  • 月2回レビュー
  • 改善提案 / 簡易チューニング
  • 優先サポート

位置づけ

  • 100〜1,000名企業向けの本命プラン
  • 「AIツール導入」ではなく、問い合わせ対応・情報検索・定型依頼の初動を業務に定着させる伴走支援として提案する

Enterprise / 全社導入プラン

  • 初期導入費: 400万円〜
  • 月額: 150万円〜
  • 契約期間目安: 12か月〜
  • 対象: 高ガバナンス要件・複数部門横断・全社標準化

含まれる内容

  • 全社導入設計 / 部門横断ユースケース整理
  • セキュリティ / 監査 / 権限設計支援
  • SSO / ログ / 承認フロー等の高度要件整理
  • 週次〜隔週の定例会
  • KPI設計 / 導入効果レポート
  • 専任窓口
  • SLA個別協議

位置づけ

  • 役員・情報システム・事業部を巻き込んだ導入を前提にしたプラン
  • 単なる上位版ではなく、社内標準として運用を成立させるための設計費が主役

5. Enterprise下限価格の考え方

Enterprise の下限は、初期400万円 / 月額150万円を基準とします。

この下限に納得感がある理由は以下です。

  1. 要件整理の関係者が増える
    情シス、管理部門、現場部門、決裁者など調整対象が一気に増える
  2. ガバナンス設計の難度が上がる
    権限、監査、ログ、SSO、承認フローなど、Business より明確に工数が増える
  3. 全社標準化では失敗コストが高い
    安価に始めて設計を誤るほうが、後戻りコストが大きい
  4. Businessとの差を明確にできる
    Enterprise を安く見せると、Business の説得力が下がる

営業上は、Enterprise の実勢レンジは月額150〜250万円として説明し、150万円は「最小構成の下限」と位置づけるのが安全です。

6. 個別開発・追加オプション

個別開発費の目安

  • 軽微カスタム: 150万円〜
  • 中規模開発: 300万〜500万円
  • 大規模開発: 800万円〜

オプション例

  • 追加ワークショップ: 20万円 / 回
  • 追加部門オンボーディング: 30万円 / 部門
  • KPIレポート強化: 25万円 / 月〜
  • 管理者 / 現場向け研修: 30万円 / 回
  • 導入診断アセスメント単体: 50万円

7. 値引きポリシー

  • 年額一括: 月額10%引きまで
  • 初年度導入キャンペーン: 初期導入費15%引きまで
  • それ以上の調整は、以下で行う
    • 定例回数を減らす
    • 対象部門数を絞る
    • 連携対象を標準範囲に限定する
    • 個別開発をフェーズ分割する

8. LP / 営業資料と揃えるべきメッセージ

全資料で統一するメッセージは以下です。

  • OpenClaw はAIツール販売ではない
  • OpenClaw は業務の初動対応・情報検索・定型依頼を仕組み化する導入伴走サービスである
  • 価値は「機能数」ではなく、現場工数の削減、導入判断のしやすさ、全社展開の再現性にある
  • まずは部門導入で成果を作り、その後に本格展開する

9. 営業判断

  • 入口商品は Pilot
  • 売上の本命は Business
  • 粗利と差別化の本命は Enterprise + 個別開発

したがって、営業では Business 80万円/月を標準提案の中心に据え、Pilot は入口、Enterprise はガバナンス要件が明確な案件に絞って提案するのが最も通しやすい構成です。
